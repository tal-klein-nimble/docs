Extract Templates are pre-built, production-ready extractors for popular platforms like Amazon, LinkedIn, Google, and more. Skip complex configuration and use simple inputs like search terms or product IDs instead of URLs.

Common uses:

- **E-commerce scraping**: Amazon, eBay, Walmart product and search data
- **Professional data**: LinkedIn profiles, job listings, company information
- **Search engines**: Google, Bing search results and knowledge panels
- **Social media**: Twitter, Facebook, Instagram public data
- **Real estate**: Zillow, Realtor listings and property details

## Two main APIs

### Discover Templates

1. Templates List
2. Template Details

Browse and inspect available templates:

```python
# List templates
templates = nimble.list_templates(
    category="ecommerce",
    page=1,
    limit=20
)

# Get template details
template_info = nimble.get_template("amazon_product")
```

### Extract with Templates

Run a template with simple inputs:

```python
result = nimble.extract_template(
    template="amazon_product",
    inputs={"asin": "B08N5WRWNW", "zipcode": "90210"}
)
```

## Quick start

### Using a template

<CodeGroup>

```python Python
from nimble import Nimble

nimble = Nimble(api_key="YOUR-API-KEY")

# Extract Amazon product data
result = nimble.extract_template(
    template="amazon_product",
    inputs={
        "asin": "B08N5WRWNW",
        "zipcode": "90210"
    }
)

print(result)
```


```node Node.js
import { Nimble } from "@nimbleway/sdk";

const nimble = new Nimble({ apiKey: "YOUR-API-KEY" });

// Extract Amazon product data
const result = await nimble.extractTemplate({
  template: "amazon_product",
  inputs: {
    asin: "B08N5WRWNW",
    zipcode: "90210"
  }
});

console.log(result);
```


```bash cURL
curl -X POST 'https://api.webit.live/api/v1/templates/extract' \
--header 'Authorization: Bearer <YOUR-API-KEY>' \
--header 'Content-Type: application/json' \
--data-raw '{
    "template": "amazon_product",
    "inputs": {
        "asin": "B08N5WRWNW",
        "zipcode": "90210"
    }
}'
```

</CodeGroup>

### Discovering templates

<CodeGroup>

```python Python
from nimble import Nimble

nimble = Nimble(api_key="YOUR-API-KEY")

# List all e-commerce templates
templates = nimble.list_templates(
    category="ecommerce",
    page=1,
    limit=20
)

for template in templates["items"]:
    print(f"{template['name']}: {template['description']}")
```


```node Node.js
import { Nimble } from "@nimbleway/sdk";

const nimble = new Nimble({ apiKey: "YOUR-API-KEY" });

// List all e-commerce templates
const templates = await nimble.listTemplates({
  category: "ecommerce",
  page: 1,
  limit: 20
});

templates.items.forEach(template => {
  console.log(`${template.name}: ${template.description}`);
});
```


```bash cURL
curl -X GET 'https://api.webit.live/api/v1/templates?category=ecommerce&page=1&limit=20' \
--header 'Authorization: Bearer <YOUR-API-KEY>'
```

</CodeGroup>

### Getting template details

<CodeGroup>

```python Python
from nimble import Nimble

nimble = Nimble(api_key="YOUR-API-KEY")

# Get full details for a template
template = nimble.get_template("amazon_product")

print("Description:", template["description"])
print("Inputs:", template["input_schema"])
print("Outputs:", template["output_schema"])
print("Domain:", template["domain"])
```


```node Node.js
import { Nimble } from "@nimbleway/sdk";

const nimble = new Nimble({ apiKey: "YOUR-API-KEY" });

// Get full details for a template
const template = await nimble.getTemplate("amazon_product");

console.log("Description:", template.description);
console.log("Inputs:", template.input_schema);
console.log("Outputs:", template.output_schema);
console.log("Domain:", template.domain);
```


```bash cURL
curl -X GET 'https://api.webit.live/api/v1/templates/amazon_product' \
--header 'Authorization: Bearer <YOUR-API-KEY>'
```

</CodeGroup>

## Templates vs Custom Extract

| Feature           | Templates            | Custom Extract            |
| ----------------- | -------------------- | ------------------------- |
| **Setup**         | Zero configuration   | Manual configuration      |
| **Maintenance**   | Maintained by Nimble | You maintain              |
| **Cost**          | Optimized pricing    | Variable based on config  |
| **Customization** | Fixed schema         | Full control              |
| **Time to value** | Immediate            | Minutes to hours          |
| **Reliability**   | Battle-tested        | Depends on implementation |
| **Best for**      | Common platforms     | Unique sites              |

**Use templates when:**

- Extracting from popular platforms
- You want zero maintenance
- Fast time to market is important
- Standard data schemas work for you

**Use custom extract when:**

- Working with unique/niche sites
- You need custom data structures
- Site requires specific configurations
- Template doesn't exist for your use case
